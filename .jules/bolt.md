## 2024-05-28 - React Leaflet Memoization Breakdown
**Learning:** Passing inline functions (like `onClick={() => setObj(u)}`) to memoized components inside a `MapContainer` completely defeats `React.memo` because the inline function reference changes every render. In React Leaflet, this is particularly damaging because it forces heavy `Marker` and `Popup` components to re-render, leading to laggy interactions when the map updates (e.g., live location tracking). Furthermore, the `eventHandlers` object itself must also be memoized using `useMemo` for React Leaflet to not re-register events on every render.
**Action:** Always pass stable references (like `setState` directly, or functions wrapped in `useCallback`) as props to memoized map components. Always memoize `eventHandlers` objects inside `Marker` components.
