## 2024-05-24 - Missing Admin Authorization & Error Leaks
**Vulnerability:** The `sendKycEmail` Cloud Function in `functions/src/index.ts` did not verify if the calling user was an ADMIN, allowing any authenticated user to trigger official KYC support emails to arbitrary users. Furthermore, multiple HTTP endpoints and callables were exposing raw `error.message` strings directly to clients, potentially leaking internal implementation details or stack trace fragments.
**Learning:** We must explicitly verify admin privileges (`role === 'ADMIN'`) for all administrative actions, even those that seem benign like sending emails. We must also sanitize error responses, failing securely with generic messages rather than propagating internal exception messages to the client.
**Prevention:** Always assert administrative privileges before executing sensitive actions in Cloud Functions. Ensure global error handlers or individual catch blocks return fixed, generic error strings instead of dynamic error messages.
