## 2024-05-18 - React Leaflet EventHandlers Defeating React.memo
**Learning:** In React Leaflet, passing an inline object or inline function for the `eventHandlers` prop on a `Marker` defeats `React.memo` entirely, even if the marker component is memoized. Furthermore, using inline arrow functions in `.map()` loops to pass down custom click handlers (e.g., `onClick={() => setSomeState(id)}`) forces the memoized wrapper component to re-render. In a map with many markers that are continuously updated, this causes massive, expensive DOM and Leaflet-level re-rendering overhead on every main Map state change.
**Action:** When working with React Leaflet markers in a `.map()` list, always pass stable references (like `useState` setters or `useCallback`s that do not depend on loop iteration variables) as props to the custom marker component. Inside the custom marker component, use `useMemo` to construct the Leaflet `eventHandlers` object, providing dependencies properly.
